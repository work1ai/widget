This task verifies the security posture and fixes any gaps found. Run these checks and fix anything that fails:

**Shadow DOM isolation (SEC-01, SEC-02):**
1. Verify all components use Lit's Shadow DOM (default behavior -- Lit components render in shadow root unless explicitly disabled). Grep for any `createRenderRoot` override that returns `this` (which would disable Shadow DOM). There should be none.
2. Verify that Plan 01 added inherited property resets to `:host` in widget.styles.ts. If any of these are missing, add them:

   - `line-height: 1.45`
   - `color: #1a1a1a`
   - `letter-spacing: normal`
   - `word-spacing: normal`
   - `text-align: left`
   - `text-transform: none`
   - `-webkit-font-smoothing: antialiased`
3. Verify no light DOM rendering (no content rendered outside shadow root).

**CSP compliance (SEC-03):**
1. Grep the entire src/ directory for `style="` (inline style attributes). There should be zero occurrences in template literals. The dynamic `<style>` element pattern (rendered inside shadow root) is CSP-safe -- it is NOT an inline style attribute.
2. Grep for `eval(`, `new Function(`, `document.write(`. There should be zero occurrences.
3. Verify that Lit uses `adoptedStyleSheets` for static styles (this is Lit's default behavior in modern browsers -- no action needed, just verify understanding).

**::part() verification (THEM-04):**
1. Grep all component files for `part="` to confirm the 5 parts are in place: bubble, panel, header, message-list, input.
