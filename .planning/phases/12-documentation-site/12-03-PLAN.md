  <action>
Create `docs/events.md` documenting the WebSocket lifecycle and DOM events (DOCS-05). Structure:

1. **Page title**: "Events & Connection"

2. **Connection Lifecycle** section:
   - Explain: WebSocket connection initiates on first panel open (not on page load)
   - Connection states table:
     | State | Status Dot | Meaning |
     |-------|-----------|---------|
     | Connected | Green (#22c55e) | WebSocket is open and ready |
     | Connecting / Reconnecting | Yellow (#eab308) | Establishing or re-establishing connection |
     | Disconnected | Red (#ef4444) | WebSocket is closed |
   - Explain the greeting behavior: when `greeting` attribute is set, the greeting message appears as the first agent bubble after WebSocket connects (not sent to the server)

3. **DOM Events** section:
   - All events bubble and are composed (cross shadow DOM boundary)
   - Event reference table:
     | Event | Detail | When |
     |-------|--------|------|
     | w1-connected | `{ session_id: string }` | WebSocket connects successfully |
     | w1-disconnected | `{ code: number, reason: string }` | WebSocket disconnects or is rejected |
     | w1-error | `{ content: string }` | Server sends an error message |
     | w1-session-end | `{ reason: string, content: string }` | Server ends the session |

4. **Listening for Events** section:
   - Show how to listen from the host page:
     ```javascript
     const widget = document.querySelector('work1-chat-widget');

     widget.addEventListener('w1-connected', (event) => {
       console.log('Connected! Session:', event.detail.session_id);
     });

     widget.addEventListener('w1-disconnected', (event) => {
       console.log('Disconnected:', event.detail.reason);
     });

     widget.addEventListener('w1-error', (event) => {
       console.error('Error:', event.detail.content);
     });

     widget.addEventListener('w1-session-end', (event) => {
       console.log('Session ended:', event.detail.reason);
     });
     ```

5. **Server URL Configuration** section:
   - The `server-url` attribute specifies the WebSocket endpoint
   - Connection only initiates when the user opens the chat panel for the first time
   - Example: `<work1-chat-widget server-url="wss://api.example.com/ws"></work1-chat-widget>`

6. **Debug Mode** section:
   - Enable with `debug` attribute to see WebSocket frames in browser console
   - Example: `<work1-chat-widget server-url="..." debug></work1-chat-widget>`

7. **Content guidelines:**
   - Code examples are vanilla JS only (no frameworks)
   - One concept per code block
   - Copy-paste ready event listener examples
