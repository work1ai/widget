**src/components/message-bubble.ts:**

Export a pure helper function `shouldGroup(messages: ChatMessage[], index: number): { isFirstInGroup: boolean; isLastInGroup: boolean }` that compares adjacent messages by role:
- `isFirstInGroup`: no previous message or previous message has different role
- `isLastInGroup`: no next message or next message has different role

Export `renderMessageBubble(message: ChatMessage, grouping: { isFirstInGroup: boolean; isLastInGroup: boolean }): TemplateResult`:
- Renders a `<div>` with classes: `message`, `message--${message.role}`
- Grouping classes based on isFirstInGroup/isLastInGroup:
  - Both true: `message--solo` (standalone bubble, full border-radius)
  - First only: `message--first` (full top radius, tight bottom margin)
  - Last only: `message--last` (pointed tail via asymmetric border-radius, tight top margin)
  - Neither: `message--middle` (reduced radius on grouped side, tight margins)
- Content: `<div class="message-content">${message.content}</div>`
- System messages (role='system'): centered, no bubble styling, muted text, smaller font

**src/styles/messages.styles.ts** -- `messageStyles`:

**Grouping styles:**
- `.message--first` -- normal top margin (8px), tight bottom margin (1px)
- `.message--middle` -- tight top and bottom margin (1px), reduced border-radius on grouped side (4px)

- `.message--solo` -- full border-radius on all sides, normal margins

Use `box-shadow: 0 1px 2px rgba(0,0,0,0.08)` on user and agent bubbles for subtle depth.

**src/components/message-list.ts:**

This file needs a slightly different pattern than pure render functions because it manages IntersectionObserver state. Export a class `ScrollManager` and a render function.

**`ScrollManager` class:**
- `isAtBottom: boolean = true` -- tracks if user is at the bottom
- `unreadCount: number = 0` -- count of messages arrived while scrolled up
- `private observer: IntersectionObserver | null = null`
- `private sentinel: HTMLElement | null = null`
- `private host: ReactiveControllerHost` -- for requestUpdate

- `setup(container: HTMLElement, sentinel: HTMLElement, host: ReactiveControllerHost)` -- create IntersectionObserver watching sentinel element with root=container, threshold=0.1. On intersection: set isAtBottom=true, reset unreadCount=0, call host.requestUpdate(). On non-intersection: set isAtBottom=false.
- `destroy()` -- disconnect observer, null references
- `onNewMessage()` -- if isAtBottom, call `this.sentinel?.scrollIntoView({ behavior: 'instant' as ScrollBehavior })`. Else increment unreadCount and call host.requestUpdate().
- `scrollToBottom()` -- call `this.sentinel?.scrollIntoView({ behavior: 'smooth' })`, reset unreadCount.

**`renderMessageList(messages: ChatMessage[], scrollManager: ScrollManager, onScrollToBottom: () => void): TemplateResult`:**
- Renders messages using `repeat` directive (from lit/directives/repeat.js) with message.id as key for stable DOM identity (prevents scroll position loss on re-render per Pitfall 2 in research)
- For each message: call `shouldGroup(messages, index)` then `renderMessageBubble(message, grouping)`
- After all messages, render a sentinel div: `<div class="scroll-sentinel"></div>` (zero height, used by IntersectionObserver)
- If `scrollManager.unreadCount > 0` AND `!scrollManager.isAtBottom`, render the scroll-to-bottom pill:
  ```html
  <button class="scroll-pill" @click=${onScrollToBottom}>
    ${scrollManager.unreadCount} new message${scrollManager.unreadCount > 1 ? 's' : ''}
  </button>
  ```
- `.scroll-pill` styles in messages.styles.ts: `position: sticky; bottom: 8px; align-self: center; background: var(--w1-accent-color, #0066FF); color: white; border: none; border-radius: 16px; padding: 6px 14px; font-size: 12px; cursor: pointer; box-shadow: 0 2px 8px rgba(0,0,0,0.15); z-index: 1;`
- `.scroll-sentinel` -- `height: 0; width: 0; overflow: hidden;`

If the panel is closed on first render (panel not visible), defer setup to when panel opens. Simplest approach: in `updated()`, check if panel is open and observer not yet initialized, then init.
