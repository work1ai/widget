```typescript
import { ReactiveController, ReactiveControllerHost } from 'lit';
import { ChatClient } from './chat-client.js';

```

**Public reactive state** (mutations call `host.requestUpdate()`):
- `messages: ChatMessage[]` -- all messages in conversation
- `connectionState: ConnectionState` -- current connection state
- `isOpen: boolean` -- panel open/close state
- `inputDisabled: boolean` -- true when disconnected/connecting

**Public methods:**
- `constructor(host: ReactiveControllerHost)` -- register with host
- `connect(url: string, debug: boolean)` -- create ChatClient, wire events, connect
- `disconnect()` -- close client, null reference, reset state
- `send(content: string)` -- validate connected, add user message (crypto.randomUUID() for id), call client.send()
- `toggleOpen(greeting?: string)` -- flip isOpen. On FIRST open, if greeting is provided and messages array is empty, add an agent message with the greeting text
- `hostConnected()` -- no-op for now
- `hostDisconnected()` -- call disconnect()

**Private method `wireClientEvents()`** -- listen on ChatClient for:
- `connected` -> connectionState='connected', inputDisabled=false
- `disconnected` -> connectionState='disconnected', inputDisabled=true
- `rejected` -> connectionState='disconnected', inputDisabled=true
- `reconnecting` -> connectionState='reconnecting'
- `session_end` -> connectionState='disconnected', inputDisabled=true, add system message with reason
- `error` -> add system message with error content
- Token, typing, message_end, status listeners: add stubs with TODO comments for Phase 3

Each state mutation MUST call `this.host.requestUpdate()`.

Per research: connect on first open, keep connection alive across panel close/reopen. Only disconnect on explicit disconnect() or hostDisconnected().
