  <action>
Create `src/work1-chat-widget.ts` -- a minimal Lit element that wraps ChatClient and exposes the public API.

**Element registration:**
- Use `@customElement('work1-chat-widget')` decorator
- Extends LitElement

**Reactive properties (Lit @property):**
- `serverUrl: string` -- mapped to `server-url` attribute via `@property({ attribute: 'server-url' })`
- `debug: boolean` -- mapped to `debug` attribute via `@property({ type: Boolean, reflect: true })`

**Private state:**
- `client: ChatClient | null = null`

**Public API methods:**
- `openConnection(): void` -- Creates new ChatClient({debug: this.debug}), calls setupClientListeners(), then calls client.connect(this.serverUrl). If serverUrl is empty, console.error('[work1-widget] server-url attribute is required') and return.
- `closeConnection(): void` -- Calls client.disconnect(), sets client to null.
- `get sessionId(): string | null` -- Returns client?.sessionId ?? null.

**setupClientListeners() -- bridges ChatClient events to DOM custom events:**
For each of these ChatClient events, add a listener that dispatches a corresponding DOM CustomEvent with `{ bubbles: true, composed: true }` (required for Shadow DOM traversal per pitfall #4):

| ChatClient Event | DOM Event Name | Detail |
|-----------------|----------------|--------|
| connected | w1-connected | { session_id } |
| disconnected | w1-disconnected | { code, reason } |
| rejected | w1-disconnected | { code, reason: 'Connection rejected' } (map rejected to disconnected for simpler public API) |
| error | w1-error | { content } |
| session_end | w1-session-end | { reason, content } |

Note: rejected maps to w1-disconnected (not a separate event) per the user decision that custom DOM events are: w1-connected, w1-disconnected, w1-error, w1-session-end (4 events).

**render() method:**
- Return `html\`<slot></slot>\`` -- minimal shell, UI rendering deferred to Phase 2
- No Shadow DOM styles yet

**No createRenderRoot override** -- let Lit use Shadow DOM by default (needed for encapsulation in later phases).
