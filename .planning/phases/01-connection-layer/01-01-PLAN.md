<task type="auto">
  <name>Task 2: Define protocol types, event map, and message validator</name>
  <files>src/chat-client.types.ts, src/index.ts</files>
  <action>
Create `src/chat-client.types.ts` with all protocol types from DRAFT.md.

**ServerMessage discriminated union** -- all 9 server message types:
- `{ type: 'connected'; session_id: string }`
- `{ type: 'token'; content: string }`
- `{ type: 'typing'; active: boolean }`
- `{ type: 'message_end' }`
- `{ type: 'session_start'; session_id: string }`
- `{ type: 'status'; content: string }`
- `{ type: 'reconnecting' }`
- `{ type: 'session_end'; reason: string; content: string }`
- `{ type: 'error'; content: string }`

**ClientMessage type:**
- `{ type: 'message'; content: string }`

**ChatClientEventMap interface** -- maps event names to CustomEvent types:
- 'connected': CustomEvent<{ session_id: string }>
- 'token': CustomEvent<{ content: string }>
- 'typing': CustomEvent<{ active: boolean }>
- 'message_end': CustomEvent<void>
- 'status': CustomEvent<{ content: string }>
- 'reconnecting': CustomEvent<void>
- 'session_end': CustomEvent<{ reason: string; content: string }>
- 'error': CustomEvent<{ content: string }>
- 'disconnected': CustomEvent<{ code: number; reason: string }> (synthetic, from WebSocket onclose for non-1000/non-1008)
- 'rejected': CustomEvent<{ code: number }> (synthetic, from WebSocket close code 1008)

**TypedEventTarget interface** -- generic wrapper for type-safe addEventListener/removeEventListener (see research doc for pattern).

**isServerMessage(data: unknown): data is ServerMessage** -- runtime type guard:
- Check data is non-null object with string `type` field
- Switch on type, validate required fields for each variant
- Return false for unknown types (supports CONN-08 graceful handling)

**Export everything** from the file. Update `src/index.ts` to re-export all from `./chat-client.types.js`.

After creating, run `npx tsc --noEmit` to verify types compile cleanly.
  </action>
