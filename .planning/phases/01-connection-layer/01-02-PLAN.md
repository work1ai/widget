  <name>ChatClient - WebSocket Protocol Client</name>
  <files>src/chat-client.ts, src/__tests__/chat-client.test.ts</files>
  <behavior>
    **Connection lifecycle:**
    - connect(url) opens WebSocket to given URL -> CONN-01
    - On server message {type: "connected", session_id: "abc"} -> emits 'connected' event with session_id detail, stores session_id -> CONN-02
    - sessionId getter returns stored session_id (null before connected)
    - connected getter returns true when WebSocket is OPEN
    - disconnect() closes WebSocket with code 1000

    **Close handling:**
    - Close code 1008 -> emits 'rejected' event with {code: 1008} -> CONN-03
    - Close code != 1000 and != 1008 -> emits 'disconnected' event with {code, reason} -> CONN-04
    - Close code 1000 (normal) -> no event emitted (intentional disconnect)
    - After any close -> ws is set to null, connected returns false

    **Protocol messages:**
    - {type: "token", content: "hello"} -> emits 'token' with {content: "hello"}
    - {type: "typing", active: true} -> emits 'typing' with {active: true}
    - {type: "typing", active: false} -> emits 'typing' with {active: false}
    - {type: "message_end"} -> emits 'message_end'
    - {type: "status", content: "Looking up..."} -> emits 'status' with {content: "Looking up..."} -> CONN-06
    - {type: "reconnecting"} -> emits 'reconnecting' -> CONN-05
    - {type: "session_end", reason: "idle_timeout", content: "Session closed..."} -> emits 'session_end' with {reason, content} -> CONN-07
    - {type: "error", content: "Something went wrong"} -> emits 'error' with {content}
    - {type: "session_start", session_id: "abc"} -> emits internally (parsed) but NOT as a public event (user decision)

    **Resilience (CONN-08):**
    - Non-JSON string -> console.warn, no crash, no event
    - JSON with unknown type -> console.warn, no crash, no event
    - JSON with missing required fields -> console.warn, no crash, no event
    - JSON object with no type field -> console.warn, no crash, no event

    **Sending:**
    - send(content) -> sends JSON {type: "message", content} to WebSocket
    - send() when not connected -> no-op (does not throw)

    **Debug mode:**
    - constructor({debug: true}) enables debug logging
    - Each received message logs parsed summary: "[work1-widget] connected session=abc123"
    - Token logs character count: "[work1-widget] token (12 chars)"
    - Default (no debug) -> silent, no console.log calls
    - Warnings (malformed messages) always log regardless of debug flag
  </behavior>
  <implementation>
ChatClient extends EventTarget (using the TypedEventTarget pattern from chat-client.types.ts).

**Constructor:** accepts optional {debug: boolean}. Stores debug flag.

**connect(url: string):** Creates new WebSocket(url). Sets onmessage -> handleMessage, onclose -> handleClose, onerror -> no-op (errors surface through onclose per pitfall #1 in research).

**disconnect():** Calls ws.close(1000, 'User closed'), sets ws to null.

**send(content: string):** If ws and ws.readyState === WebSocket.OPEN, send JSON.stringify({type: 'message', content}).

**handleMessage(event: MessageEvent):**
1. Try JSON.parse(event.data), catch -> warn and return
2. Use isServerMessage() type guard. If false -> warn about unknown/malformed type and return
3. Switch on data.type, dispatch appropriate CustomEvent
4. For 'connected': also store session_id on instance
5. For 'session_start': parse but do NOT dispatch (user decision -- not surfaced to public API)
6. If debug: log parsed summary before dispatching

**handleClose(event: CloseEvent):**
1. If code === 1008 -> dispatch 'rejected' event
2. Else if code !== 1000 -> dispatch 'disconnected' event
3. Set ws to null, reset session_id to null

**Properties:**
- get sessionId(): string | null
- get connected(): boolean (ws?.readyState === WebSocket.OPEN)
