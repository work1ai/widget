  <action>
Per research, Rollup's external config applies to ALL outputs in a single build. The user decision requires ESM to externalize deps and IIFE to bundle everything. Solution: two separate Vite configs with two build scripts.

**vite.config.ts** -- Replace contents. This becomes the ESM build config:
```typescript
import { defineConfig } from 'vite';
import dts from 'vite-plugin-dts';

export default defineConfig({
  plugins: [
    dts({ include: ['src'] }),
  ],
  build: {
    lib: {
      entry: 'src/index.ts',
      formats: ['es'],
      fileName: () => 'work1-chat-widget.es.js',
    },
    rollupOptions: {
      external: [
        'lit',
        /^lit\//,
        'marked',
        'dompurify',
      ],
    },
    emptyOutDir: true,
  },
});
```

**vite.config.iife.ts** -- Create new file. Self-contained IIFE build (no externals, no dts plugin needed):
```typescript
import { defineConfig } from 'vite';

export default defineConfig({
  build: {
    lib: {
      entry: 'src/index.ts',
      name: 'Work1ChatWidget',
      formats: ['iife'],
      fileName: () => 'work1-chat-widget.iife.js',
    },
    emptyOutDir: false,
  },
});
```

`emptyOutDir: false` on IIFE build so it does not delete the ESM output from the first build. ESM build runs first with `emptyOutDir: true` to clean.

**scripts/report-size.cjs** -- Create new file. Post-build script that prints raw and gzipped sizes of the IIFE bundle:
```javascript
const fs = require('fs');
const zlib = require('zlib');
const path = require('path');

const file = path.join(__dirname, '..', 'dist', 'work1-chat-widget.iife.js');

if (!fs.existsSync(file)) {
  console.error('IIFE bundle not found at', file);
  process.exit(1);
}

const raw = fs.statSync(file).size;
const gzipped = zlib.gzipSync(fs.readFileSync(file)).length;

console.log('\nIIFE bundle size:');
console.log(`  Raw:    ${(raw / 1024).toFixed(1)} KB`);
console.log(`  Gzip:   ${(gzipped / 1024).toFixed(1)} KB\n`);
```
