Create a `<playground-controls>` Lit web component in `playground/controls.ts`. This is playground-only code, never shipped to production. Use `(widget as any).handleNewConversation()` for reconnection per research recommendation.

**Component structure:**
- `@customElement('playground-controls')` extending LitElement
- Property: `widgetEl` set externally after DOM ready (the `<work1-chat-widget>` reference)
- `@state()` properties: `connectionStatus` ('connected' | 'connecting' | 'disconnected'), `mockMode` (boolean, default true), `realUrl` (string), `expandedSections` (object tracking accordion states, all true by default)

**Styles (static styles):**
- Sidebar-internal styles: padding, section spacing, labels, control grid
- Accordion via `<details open>` and `<summary>` -- style summary as section headers with bold text, cursor pointer
- Color picker inputs: native `<input type="color">` at ~36x36px
- Control rows: label on left, input on right (flexbox row)
- Toggle switch: pure CSS checkbox toggle (44x24px, slider with circle, checked = blue #0066FF)
- Status dot: 8x8px inline circle next to Connection header text
- Scenario buttons: full-width within section, 8px gap, content buttons with neutral/blue tint (#e8f0fe bg, #1a73e8 text), error buttons with red/orange tint (#fce8e6 bg, #c5221f text)
- Reset buttons: small, text-style, right-aligned per section
- Disabled state for scenario buttons: opacity 0.4, pointer-events none
- Reconnect button: appears when disconnected, styled like a scenario content button

**Theme Section (`<details open>`):**
- Summary text: "Theme"
- Color pickers for all 10 CSS custom properties listed in the interfaces block. Each as a labeled row: label text + `<input type="color" value="${default}">`. On change, call `this.widgetEl.style.setProperty(propName, e.target.value)`.
- Position: two radio buttons labeled "Bottom Right" (default checked) and "Bottom Left". On change, call `this.widgetEl.setAttribute('position', value)`.
- Width: `<input type="range" min="300" max="500" value="380" step="10">` with value display. On change, call `this.widgetEl.setAttribute('width', value + 'px')`.
- Height: `<input type="range" min="400" max="700" value="560" step="10">` with value display. On change, call `this.widgetEl.setAttribute('height', value + 'px')`.
- Bubble icon: `<input type="text" placeholder="e.g. help-circle">`. On change, call `this.widgetEl.setAttribute('bubble-icon', value)`.
- Reset button: resets all theme controls to defaults. For each color, sets control value AND calls `widgetEl.style.removeProperty(prop)`. For position/width/height/bubble-icon, removes attributes. Call `this.requestUpdate()` to refresh control values.

**Connection Section (`<details open>`):**
- Summary text: "Connection" + status dot (green #22c55e = connected, yellow #eab308 = connecting, red #ef4444 = disconnected)
- Toggle switch labeled "Mock" / "Real" (mock = default checked). On toggle:
  1. Set `this.connectionStatus = 'connecting'`
  2. If switching to mock: set `widgetEl._wsConstructor = MockWebSocket`, set `widgetEl.setAttribute('server-url', 'ws://mock')`
  3. If switching to real: set `widgetEl._wsConstructor = undefined`, if `this.realUrl` is filled set `widgetEl.setAttribute('server-url', this.realUrl)`
  4. Call `(this.widgetEl as any).handleNewConversation()` to disconnect/clear/reconnect
- URL input: `<input type="text" placeholder="ws://localhost:8080">` visible only when `!this.mockMode`. On change, store value in `this.realUrl`. If real mode is active and URL changes, update `widgetEl.setAttribute('server-url', this.realUrl)` and trigger reconnect.
- Reconnect button: visible only when `this.connectionStatus === 'disconnected'`. On click, set status to 'connecting', call `(this.widgetEl as any).handleNewConversation()`.

**Connection state observation (in connectedCallback or when widgetEl is set):**
- Listen for `w1-connected` event on widgetEl -> set `this.connectionStatus = 'connected'`
- Listen for `w1-disconnected` event on widgetEl -> set `this.connectionStatus = 'disconnected'`

**Scenarios Section (`<details open>`):**
- Summary text: "Scenarios"
- All buttons disabled (opacity + pointer-events) when `!this.mockMode`
- Content group label "Content" with buttons:
  - "Greeting" -> `MockWebSocket.instance?.triggerScenario('greeting')`
  - "Long Markdown" -> `MockWebSocket.instance?.triggerScenario('long-markdown')`
- Error/State group label "Error / State" with buttons:
  - "Protocol Error" -> `MockWebSocket.instance?.triggerScenario('error-protocol')`
  - "Connection Rejected" -> `MockWebSocket.instance?.triggerScenario('error-rejected')`
  - "Unexpected Disconnect" -> `MockWebSocket.instance?.triggerScenario('error-disconnect')`
  - "Session End" -> `MockWebSocket.instance?.triggerScenario('session-end')`

Import `MockWebSocket` from `./mock-ws.ts`. Import standard Lit decorators from `lit` and `lit/decorators.js`.
