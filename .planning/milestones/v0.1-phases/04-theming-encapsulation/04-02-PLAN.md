**bubble-button.ts:**
Update renderBubble to accept an optional iconName parameter and use a named slot with attribute-driven fallback:

```typescript
import { getLucideIcon } from './icons.js';

export function renderBubble(
  onClick: () => void,
  position: string,
  hidden: boolean = false,
  iconName: string = '',
): TemplateResult {
  const lucideIcon = iconName ? getLucideIcon(iconName) : null;
  const fallbackIcon = lucideIcon ?? chatBubbleIcon;

  return html`
    <button
      class="bubble-button bubble--${position}${hidden ? ' bubble--hidden' : ''}"
      part="bubble"
      aria-label="Open chat"
      @click=${onClick}
    >
      <slot name="bubble-icon">
        ${fallbackIcon}
      </slot>
    </button>
  `;
}
```

The native `<slot>` mechanism handles precedence automatically: when a customer provides `<span slot="bubble-icon"><img src="..."></span>`, the slot content replaces the fallback. When no slot content is provided, the attribute-driven icon (or default chatBubbleIcon) shows.
