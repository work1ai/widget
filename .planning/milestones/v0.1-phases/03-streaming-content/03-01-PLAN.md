  <name>Task 2: Wire ChatStore streaming event handlers</name>
  <files>src/chat-store.types.ts, src/chat-store.ts</files>
  <action>
1. Update `src/chat-store.types.ts`:
   - No changes to ChatMessage (streaming field already exists)
   - Types are sufficient as-is; ChatStore will use plain properties for statusText and typingActive (not on ChatMessage)

2. Update `src/chat-store.ts` — add streaming state and event handlers:

   **New private fields:**
   - `private streamingMessageId: string | null = null`
   - `private streamingContent = ''`

   **New public reactive fields (read by UI):**
   - `statusText = ''`
   - `typingActive = false`

   **In wireClientEvents(), replace the TODO comment with four listeners:**

   a) `token` listener:
      - Set `this.typingActive = false` (first token replaces typing indicator)
      - Set `this.statusText = ''` (STRM-06: clear status on token)
      - If `!this.streamingMessageId`: create new streaming message — generate UUID, set `streamingMessageId`, set `streamingContent = e.detail.content`, append new ChatMessage with `{ id, role: 'agent', content: streamingContent, timestamp: Date.now(), streaming: true }` to messages array
      - If `this.streamingMessageId` exists: append `e.detail.content` to `streamingContent`, update message in array via `.map()` replacing the matching message's content with new `streamingContent`
      - Call `this.host.requestUpdate()`

   b) `typing` listener:
      - Set `this.typingActive = e.detail.active` (STRM-02, STRM-03)
      - Call `this.host.requestUpdate()`

   c) `message_end` listener:
      - If `this.streamingMessageId`: map over messages, set `streaming: false` on matching message
      - Reset: `this.streamingMessageId = null`, `this.streamingContent = ''`
      - Set `this.typingActive = false`
      - Set `this.statusText = ''` (STRM-06: clear status on message_end)
      - Call `this.host.requestUpdate()`

   d) `status` listener:
      - Set `this.statusText = e.detail.content` (STRM-05: only latest shown)
      - Call `this.host.requestUpdate()`

   **Update send() method** — handle concurrent streaming (user decision):
   - At the start of `send()`, if `this.streamingMessageId` is set: finalize current streaming message (set streaming: false via map), reset `streamingMessageId` and `streamingContent` and `typingActive` and `statusText`
   - Then proceed with normal send logic (add user message, call client.send)

   Use `as EventListener` cast pattern consistent with existing session_end/error listeners.
