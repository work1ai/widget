### Integration Gaps (Non-Critical)

**INT-01: sessionId not exposed to host page**
- `src/work1-chat-widget.ts:121-124` — `sessionId` getter returns `null`
- `src/work1-chat-widget.ts:312-316` — `w1-connected` event dispatches `session_id: null`
- ChatClient stores session_id correctly, but ChatStore doesn't expose it upward
- Affected: CONN-02 (external API only; internal pipeline works)

**INT-02: w1-error and w1-session-end DOM events not dispatched**
- `src/work1-chat-widget.ts:336-341` — `setupDOMEventForwarding()` is empty
- Error and session_end events update ChatStore (system messages render correctly)
- Host page `addEventListener('w1-error', ...)` would never fire
- Affected: ERR-01..03 (external event contract only)

Neither gap affects the end-user chat experience. Both affect the programmatic host-page integration API.
