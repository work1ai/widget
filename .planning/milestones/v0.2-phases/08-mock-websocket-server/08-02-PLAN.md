**File 2: `playground/mock-ws.ts`**

Create `MockWebSocket` class that implements the WebSocket interface subset used by ChatClient.

**Static constants:**
```typescript
static readonly CONNECTING = 0;
static readonly OPEN = 1;
static readonly CLOSING = 2;
static readonly CLOSED = 3;
static instance: MockWebSocket | null = null;
```

**Properties (matching WebSocket interface):**
```typescript
readyState = MockWebSocket.CONNECTING;
onmessage: ((event: MessageEvent) => void) | null = null;
onclose: ((event: CloseEvent) => void) | null = null;
onerror: (() => void) | null = null;
private timers: ReturnType<typeof setTimeout>[] = [];
```

**Constructor(url: string):**
- Store url (not used, but matches WebSocket constructor signature)
- Set `MockWebSocket.instance = this`
- After ~50ms async delay (setTimeout):
  1. Set `readyState = MockWebSocket.OPEN`
  2. Emit `{ type: 'connected', session_id: 'mock-session-001' }` via `onmessage`
  3. After another short delay (~100ms), auto-trigger the greeting scenario

**emit(msg: ServerMessage) private method:**
- Check `readyState === MockWebSocket.OPEN` before emitting (guard against post-close)
- Wrap in `new MessageEvent('message', { data: JSON.stringify(msg) })`
- Call `this.onmessage(event)` if assigned

**delay(ms: number) private method:**
- Returns `Promise<void>` using setTimeout
- Pushes timer ID to `this.timers` array for cleanup tracking

**streamResponse(content: string) private async method:**
- Emit `{ type: 'typing', active: true }`
- Await delay(200) -- typing indicator pause
- Split content on spaces into words array
- For each word: emit `{ type: 'token', content: token }` where first token has no leading space, subsequent tokens have leading space: `(i === 0 ? '' : ' ') + words[i]`
- Await delay(40) between tokens
- After all tokens: emit `{ type: 'message_end' }`

**send(data: string) public method:**
- Guard: if `readyState !== MockWebSocket.OPEN`, return
- Parse JSON from data
- If `msg.type === 'message'`, call `streamResponse('You said: ' + msg.content)` (MOCK-01 echo)
- Wrap in try/catch, ignore parse errors

**close(code?: number, reason?: string) public method:**
- Clear all pending timers: `this.timers.forEach(id => clearTimeout(id))`
- Set `readyState = MockWebSocket.CLOSED`
- If `this.onclose`, call with `new CloseEvent('close', { code: code ?? 1000, reason: reason ?? '', wasClean: true })`

**simulateClose(code: number, reason: string) private method:**
- Clear all pending timers
- Set `readyState = MockWebSocket.CLOSED`
- If `this.onclose`, call with `new CloseEvent('close', { code, reason, wasClean: false })`

**triggerScenario(name: string) public method:**
- `'greeting'`: call `streamResponse(GREETING_TEXT)` (MOCK-02)
- `'long-markdown'`: call `streamResponse(LONG_MARKDOWN_TEXT)` (MOCK-03)
- `'error-protocol'`: emit `{ type: 'error', content: ERROR_TEXT }` (MOCK-04)
- `'error-rejected'`: call `simulateClose(1008, 'Connection rejected')` (MOCK-04)
- `'error-disconnect'`: call `simulateClose(1006, 'Unexpected disconnect')` (MOCK-04)
- `'session-end'`: emit `{ type: 'session_end', reason: 'timeout', content: SESSION_END_TEXT }` (MOCK-05)

**Exports:** `export { MockWebSocket }`

**Import from scenarios.ts:** `import { GREETING_TEXT, LONG_MARKDOWN_TEXT, ERROR_TEXT, SESSION_END_TEXT } from './scenarios.ts';`

**Import protocol types for type safety:** `import type { ServerMessage } from '../src/chat-client.types.ts';`

```html
<script type="module">
  import '../src/index.ts';
  import { MockWebSocket } from './mock-ws.ts';

  const widget = document.querySelector('w1-chat-widget');
  widget.setAttribute('server-url', 'ws://mock');
  widget.setAttribute('debug', '');
  widget._wsConstructor = MockWebSocket;

  // Expose for console-based scenario triggering (Phase 9 adds UI controls)
  window.MockWebSocket = MockWebSocket;

  console.log('[playground] Mock mode active. Trigger scenarios via:');
  console.log('  MockWebSocket.instance.triggerScenario("long-markdown")');
  console.log('  MockWebSocket.instance.triggerScenario("error-protocol")');
  console.log('  MockWebSocket.instance.triggerScenario("error-rejected")');
  console.log('  MockWebSocket.instance.triggerScenario("error-disconnect")');
  console.log('  MockWebSocket.instance.triggerScenario("session-end")');
</script>
```

This ensures:
1. Widget has a server-url so it will attempt connection on first panel open
2. Debug mode is on so protocol messages are logged to console
3. `_wsConstructor` is set BEFORE the user opens the panel, so when `handleOpen()` fires, it passes MockWebSocket to `store.connect()`
4. The greeting scenario auto-fires on connect, so the developer sees a streamed greeting immediately after clicking the bubble
5. `MockWebSocket` is available on `window` for console-based scenario triggering
6. Console prints a guide showing available scenario commands
